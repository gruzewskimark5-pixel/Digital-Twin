## 2024-05-24 - React useState Lazy Initialization\n**Learning:** In a heavily-ticking React application (like this telemetry dashboard with 1-second ticks), complex functions passed directly to `useState(computeIntensiveData())` execute on *every single render*, causing unnecessary garbage collection and CPU cycles even though React discards the result after the initial render.\n**Action:** Always use lazy initialization `useState(() => computeIntensiveData())` for dynamically generated initial state arrays or expensive computations, especially in components with frequent state updates.
